# フィードバック：GAS認証未完了による「ネットワークエラー」繰り返し発生

## 結論

「ネットワークエラー」の正体は **GASスクリプトの認証未完了（403エラー）** だった。
Content-Typeの問題でもGAS URLの問題でもなかった。

---

## なぜなぜ分析（5回）

**なぜ1：** なぜスマホから保存できなかったか？
→ GASのエンドポイントが403（アクセス拒否）を返したため

**なぜ2：** なぜ403が返ったか？
→ GASスクリプトがSpreadsheet・DriveへのOAuthアクセス許可を完了していなかったため

**なぜ3：** なぜ認証が未完了だったか？
→ 再デプロイを繰り返す中で認証状態がリセットされ、`testAuth()` を実行して再許可するステップが漏れたため

**なぜ4：** なぜ認証漏れに気づかなかったか？
→ エラーメッセージが「ネットワークエラー」だけで、GASの認証問題であることを示す情報がなかったため

**なぜ5：** なぜ同じミスを繰り返したか？
→ デプロイ後に「ちゃんと動くか確認する」フローが存在せず、毎回本番のスマホ操作で初めてエラーに気づく構造になっていたため

---

## 根本原因

**GASデプロイ後の動作確認フローが存在しない。**
デプロイするたびにスマホで実際に試して初めてエラーに気づく構造になっており、認証切れ・設定ミスの発見が遅れる。

---

## 対策：デプロイ後の確認チェックリスト

GASを新規デプロイ・再デプロイしたら必ず以下を実施する：

- [ ] **① testAuth() をGASエディタから実行する**（「新しいデプロイ」を新規作成したときのみ。「新しいバージョン」での更新は不要）
- [ ] **② ブラウザでGAS URLに直接アクセスして `{"status":"running"}` が返るか確認する**（毎回実施）
- [ ] **③ スマホのSafariから保存テストを1件行う**（毎回実施）

①②を先にやれば、スマホで試す前に問題を発見できる。

---

## 実施済み修正（2026-05-05）

| 修正内容 | 効果 |
|---------|------|
| testAuth() を実行して認証を完了 | GASが正常に応答するようになった（403→200） |
| 新しいバージョンで再デプロイ | 認証状態をリセット・再適用 |
| gasPost() のContent-Typeを `text/plain;charset=utf-8` に変更 | CORSプリフライトを回避（今後の予防策） |
| ネットワークエラー時のメッセージを具体化 | 次回の自己診断をしやすくした |

## 教訓

- **「ネットワークエラー」はGASの認証切れでも発生する。**  
  XHRはHTTP 403もCORSエラーも同じ `onerror` で受け取るため、表示上区別できない。

- **再デプロイ後は必ず testAuth() を実行する習慣をつける。**  
  GASは「自分として実行」設定でも、再デプロイ後に認証のやり直しが必要になることがある。

- **デプロイ後の確認をスマホ実機に頼るな。**  
  ブラウザからGAS URLに直接アクセスすれば10秒で正常動作を確認できる。

## 追記：2026-05-18 — 「長期非アクセス」でも403になる事象

### 現象
- 再デプロイは一切していない
- 「アクセスできるユーザー＝全員」「次のユーザーとして実行＝自分」のデプロイ設定も正しい
- OAuthスコープも有効（testAuth実行で認証ダイアログが出ずに成功）
- **にもかかわらず約1ヶ月の非アクセス後に外部リクエストが 403 Forbidden を返した**
- レスポンスボディはGoogleの「アクセスが拒否されました」HTML

### 復旧手順（実測）
1. GASエディタで `testAuth()` を1回実行（認証ダイアログは出ない）
2. 直後に外部からpingを叩くと **200 OK に復活**

### 仮説（観察ベース・Google公式仕様には未確認）
所有者のスクリプト実行が長期間ない場合、外部匿名アクセス用の実行コンテキストが「休眠状態」になる可能性がある。所有者自身が一度スクリプトを実行することで再アクティブ化される。

### 根本対策（2026-05-18 実装）
**`keepAlive()` 関数と時刻トリガー** を [コード.gs](../コード.gs) に追加。  
毎日午前3時に所有者として軽量関数を自動実行することで、スクリプトを休眠させない。
- `keepAlive()` ：スプシ・ドライブに軽くアクセスするだけの関数
- `setupKeepAliveTrigger()` ：トリガー設定用（GASエディタから1回だけ実行）

### 確認すること
- [ ] GASエディタで `setupKeepAliveTrigger` を1回実行（トリガー登録）
- [ ] 「トリガー」画面で keepAlive が毎日3時に登録されているか確認
- [ ] 1ヶ月以上経過してもアプリが403にならないか観察
