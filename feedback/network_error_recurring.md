# フィードバック：GAS保存の「ネットワークエラー」繰り返し発生

## 現象

```
[INFO] 保存開始: "書籍タイトル"
[ERROR] 保存に失敗しました: ネットワークエラー
```

- Claude API（表紙解析）は成功するのに、GAS保存だけ失敗する
- 1〜2秒という短時間で失敗する（タイムアウトではない）
- スマホのブックマーク（PWA）から使用時に発生

---

## なぜ同じエラーが繰り返すのか（根本原因）

### なぜ1：保存時にネットワークエラーが出た
→ `gasPost()` の `xhr.onerror` が発火した（HTTP応答が返る前に接続失敗）

### なぜ2：接続が失敗した
→ GAS WebアプリのURLが無効になっていた、またはCORSエラーで拒否された

### なぜ3：無効なURLのまま使い続けた
→ アプリはlocalStorageに保存したGAS URLを起動時に検証せず、保存ボタンを押して初めてエラーが発覚する仕組みになっている

### なぜ4：エラーで気づけなかった
→ エラーメッセージが「ネットワークエラー」だけで、「GAS URLが古い可能性」「デプロイ設定の確認方法」「次のアクション」が示されない

### なぜ5：同じ問題が繰り返す
→ GASを再デプロイするとURLが変わるが、**アプリ側にはURLが変わったことを検知する仕組みがない**。ユーザーが手動でURLを更新しない限り、古いURLのまま使い続ける

---

## 対策（実装すべき改善）

### 対策1：アプリ起動時にGAS URLの疎通テストを実行する
- localStorageにGAS URLが設定されている場合、起動時に `?action=ping` でGETを叩く
- 失敗した場合は「⚠ GAS URLが応答しません。URLを再設定してください」を目立つ位置に表示

### 対策2：エラーメッセージに診断情報と次のアクションを追加する
現在：`保存に失敗しました: ネットワークエラー`
改善後：`保存に失敗しました: GAS URLが応答しません。①URLが正しいか ②デプロイが「全員（匿名）」になっているか 確認してください`

### 対策3：GAS側に `ping` アクションを追加する
```javascript
// コード.gs の doGet に追加
if (e.parameter.action === 'ping') return buildResponse({ ok: true });
```

### 対策4：GASを再デプロイしたらURLが変わることをDESIGN.mdに明記する
- 「バージョンを選択」で「新しいバージョン」を選ぶとURLは変わらない
- 「新しいデプロイ」を作成するとURLが変わる → アプリのURL再設定が必要

---

## GASデプロイ時の必須チェックリスト

デプロイ・再デプロイ後に必ず確認すること：

- [ ] アクセスできるユーザー：**全員（匿名ユーザーを含む）**
- [ ] 実行するユーザー：**自分**
- [ ] URLが変わった場合：アプリの「GAS WebアプリURL」欄を再設定
- [ ] `testAuth()` をGASエディタから実行して認証確認
